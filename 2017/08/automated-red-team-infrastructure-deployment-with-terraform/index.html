Automated Red Team Infrastructure Deployment with Terraform

22 August 2017 in blog

Introduction

Deploying resiliant Red Team infrastructure can be quite a time consuming process. This wiki maintained by Steve Borosh and Jeff Dimmock is probably the best public resource I’ve seen in regards to design considerations and hardening tips.

For someone like myself, who destroys and stands fresh infrastructure up for each engagement, building everything by hand is a long, laborious process. Anything that can be automated is a good thing.

This post is to provide examples of how Terraform can be used to automate Red Team infrastrucure deployment. See this GitHub repo for a full set of configuration files.

Terraform

Terraform is an open source tool that codifies APIs into declarative configuration files - allowing you to design aspects of your cloud infrastructure and deploy on demand.

Terraform supports many different providers, including AWS, Azure, Bitbucket, Cloudflare, DigitalOcean, Docker, GitHub, Google Cloud, OpenStack, OVH and vSphere to name a few.

Defining Custom Variables

variables.tf contains a list of variable names that Terraform needs to know about.

variable "token" {}
variable "ssh-key" {}
variable "attacker-ip" {}

terraform.tfvars contains the actual values for those variables.

token = "abcdefghijklmnopqrstuvwxyz0123456789"

ssh-key = "ssh-rsa AAAA[...snip...]cQ=="
or
ssh-key = "${file("/path/to/key")}"

attacker-ip = "1.2.3.4/5"

Defining Providers

providers.tf

provider "digitalocean" {
    token = "${var.token}"
}

SSH Keys

ssh-keys.tf

resource "digitalocean_ssh_key" "c2" {
    name = "c2-key"
    public_key = "${var.c2-key}"
}

Droplets / VMs

droplets.tf

resource "digitalocean_droplet" "c2-http" {
    image = "ubuntu-14-04-x64"
    name = "c2-http"
    region = "lon1"
    size = "512mb"
    ssh_keys = ["${digitalocean_ssh_key.c2.id}"]
}

Firewalls

firewalls.tf

resource "digitalocean_firewall" "c2-http" {
    name = "c2-http-fw"

    droplet_ids = ["${digitalocean_droplet.c2-http.id}"]

    inbound_rule = [
    {
        protocol = "tcp"
        port_range = "22"
        source_addresses = ["${var.attacker-ip}"]
    },
    {
        protocol = "tcp"
        port_range = "80"
        source_droplet_ids = ["${digitalocean_droplet.r1.id}"]
        source_droplet_ids = ["${digitalocean_droplet.r2.id}"]
        source_droplet_ids = ["${digitalocean_droplet.r3.id}"]
    },
    {
        protocol = "tcp"
        port_range = "443"
        source_droplet_ids = ["${digitalocean_droplet.r1.id}"]
        source_droplet_ids = ["${digitalocean_droplet.r2.id}"]
        source_droplet_ids = ["${digitalocean_droplet.r3.id}"]
    },
    {
        protocol = "tcp"
        port_range = "50050"
        source_addresses = ["${var.attacker-ip}"]
    }
    ]

    outbound_rule = [
    {
        protocol = "udp"
        port_range = "53"
        destination_addresses = ["0.0.0.0/0"]
    },
    {
        protocol = "tcp"
        port_range = "80"
        destination_addresses = ["0.0.0.0/0"]
    },
    {
        protocol = "tcp"
        port_range = "443"
        destination_addresses = ["0.0.0.0/0"]
    }
    ]
}
